리눅스 권한
소유자 권한 / 소유그룹권한 / 기타유저권한
- / rw- / rw- / r--

rwx : 모든권한
r : 읽기 권한
w : 쓰기 권한
x : 실행 권한

groups : 그룹 보는 명령어
groups [그룹이름] : 속한 그룹 보기

권한 변경 방법
chmod(change mode) 명령어

기호 모드
chmod u(소유자)/g(그룹)/o(기타)/a(모든사용자) + (권한(r/w/x)) "파일 및 폴더명" : 권한 추가
chmod u(소유자)/g(그룹)/o(기타)/a(모든사용자) - (권한(r/w/x)) "파일 및 폴더명" : 권한 제거
chmod g-rw : 모든권한 다 빼앗기
chmod o-r,g+rw :복합적으로도 가능함
chmod og+x 

숫자 모드
chmod 744 파일명
rwxr--r--

change u+rwx, g+rw, o+r express2
chmod 764 express2

특수 접근 권한

777 -> 일반 권한(특수 권한이 숨겨져잇음)
0002 -> 특수 + 일반 권한

setuid -> 파일 소유자 권한으로 실행
파일 소유자가 : A
실행 주체 : B
B가 실행 했는데, A가 실행한 것처럼 실행됨

setgid -> 파일 소유 그룹 권한으로 실행됨
파일 소유 그룹 : A
실행 주체 그룹 : B
B 그룹이 실행 했는데, A 그룹이 실행한 것처럼 실행됨

sticky bit
- 디렉터리에만 설정 할 수 있음
- 스티키 비트가 설정된 디렉터리는 누구나 파일 생성가능
- 생성된 파일은 실행 객체 소유로 설정됨.
- 공유 폴더 <- 스티키 비트 특수 권한 설정
- A, B, C가 공유 폴더에 파일을 만듬
- a.txt의 소유자는 A, B.txt의 소유자는 B ...
- B가 a.txt를 수정하려고 함.

일반 권한 부여
chmod 777 파일명

특수 권한 부여
chmod 4777 파일명
4 -> setuid
2 -> setgid
1 -> sticky bit

-rws로 특수 권한이 표현된다
-rw 's' r : setuid
-rwxrw 's' r : setgid
--r'wt' : sticky bit
