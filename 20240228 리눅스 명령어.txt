1. nginx 패키지를 설치합니다
sudo apt-get install nginx

2. nginx 패키지가 설치한 파일 목록을 검색하고, 결과를 nginx-installed.log 파일 내용에 저장합니다.
sudo dpkg -L nginx
sudo dpkg -L nginx > nginx-installed.log

3. nginx 패키지를 설정 파일을 포함하여 삭제합니다.
sudo apt-get remove --purge nginx

4. nginx 패키지 설치 파일을 ubuntu 유저 홈 디렉터리에 다운로드 받습니다.
sudo apt-get download nginx

5. nginx 패키지 설치 파일로 패키지를 다시 설치합니다.
sudo dpkg -i nginx

6. 다시 nginx 패키지를 설정 파일 포함하여 삭제합니다.
sudo dpkg --purge nginx

프로세스 (task, service)
: 컴퓨터에서 실행중인 프로그램
	(실행중이면 프로세스, 실행중이지 않으면 프로그램)

프로그램
: 하드디스크에 저장되어 있는 실행코드

프로그램을 실행하면
-> 하드디스크에서 메모리로 임시 복사해서 이동(캐싱)

백그라운드 프로세스
: 명령 처리가 끝나는 것과 관계없이 다른 작업이 가능(비동기)
포그라운드 프로세스
: 명령 처리가 끝날때 까지 다른 작업 불가(동기)

부모 프로세스
자식 프로세스
- 부모 프로세스가 생성한다.
- 자식 프로세스를 생성할 수 있다.
- 할 일이 끝나면 부모 프로세스에 결과를 돌려주고 종료한다.


프로세스 종류
- 데몬 프로세스
	: 실행 후 결과에 상관없이 계속 실행되는 프로세스
	  리눅스에서는 리눅스 커널에 의해 실행됨

- 고아 프로세스
	: 자식 프로세스가 실행중인데 부모 프로세스가 먼저 종료된 프로세스
	  고아 프로세스가 부모 프로세스가 되어 연관된 고아 프로세스를 작업 종료

- 좀비 프로세스
	: 자식 프로세스가 실행중인데 부모 프로세스가 먼저 종료된 프로세스
	  고아 프로세스가 종료가 안되어서 프로세스 테이블 목록에 남아있으면 좀비
	  좀비 프로세스가 너무 많으면 메모리 용량이 부족해서 일반 프로세스가 실행되지 않을 수도 잇다.
	  
프로세스 상태
- 생성(create)
	: 프로세스가 생성되는 중
- 실행(running)
	: 프로세스 실행 중
- 준비(ready)
	: 현재 CPU를 사용하고 있지 않음
	  CPU가 활당되기를 기다리는 중
- 대기 / 보류(waiting / block)
	: 프로세스가 입출력이 완료 등을 기다리고 있는 상태
- 종료(terminated/dead)
	: 프로세스 종료.
